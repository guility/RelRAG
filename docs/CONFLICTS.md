# Реестр конфликтов при разработке

При возникновении **нестыковки результата текущей задачи с результатами предыдущих этапов** необходимо:

1. **Зафиксировать конфликт** в этом файле (шаблон ниже)
2. **Вернуть задачу на доработку** автору задачи, с которой возник конфликт
3. После доработки **автор фиксирует решение** в той же записи конфликта

---

## Правило эскалации

- **1-й и 2-й возврат**: автор дорабатывает задачу, фиксирует решение в CONFLICTS.md
- **3-й возврат**: привлекается **архитектор** для анализа причин; вносятся исправления в задачи/контракты; решения фиксируются в [CONVENTIONS.md](CONVENTIONS.md) и в ADR (`docs/adr/`)

---

## Шаблон записи конфликта

```markdown
### CONFLICT-YYYY-MM-DD-NNN

| Поле | Значение |
|------|----------|
| **Дата обнаружения** | YYYY-MM-DD |
| **Обнаруживший** | Имя/ник |
| **Задача с конфликтом** | T2.1 (или другая) |
| **Автор задачи** | Имя/ник |
| **Этап, с которым конфликт** | T1.4 (структура папок) |
| **Описание конфликта** | Что именно не стыкуется |
| **Возвратов** | 1 / 2 / 3 |

#### Решение (заполняет автор после доработки)

| Поле | Значение |
|------|----------|
| **Дата решения** | YYYY-MM-DD |
| **Принятое решение** | Как конфликт был устранён |
| **Изменённые файлы/задачи** | Список |

#### Эскалация (если 3-й возврат)

| Поле | Значение |
|------|----------|
| **Архитектор** | Имя |
| **Анализ причин** | Краткое описание |
| **Изменения в CONVENTIONS.md** | Ссылка на раздел |
| **ADR** | Номер ADR |
```
